DEPLOYMENT
 * Projektet compileras som 64-bitars
 * Refererade assemblys ska ej kopieras lokalt.
   References > [resp. assembly] > Properties > Copy Local = False
 * Ändring på plats i Settings.config kräver en ändring och omsparning av web.config (mellanslag i filen och spara).
   Annars tas inte ändringen.
 * Stoppa berörd applikationspool
 * Rensa ev. ASP.NET cache under Microsoft .Net Framework
   För 64-bitars:
   C:\Windows\Microsoft.NET\Framework64\v4.0.30319\Temporary ASP.NET Files
   För 32-bitars
   C:\Windows\Microsoft.NET\Framework\v4.0.30319\Temporary ASP.NET Files
 * Backup webbapplikation
 * Radera webbapplikationen så när som för
    + filen Settings.config (om inga ändringar gjorts)
    + Katalogen .\log
 * Kopiera nya filer till skarp katalog
 * Kopiera äldre binärer från backup men skriv inte över filerna som frågas för överskrivning
   (de som önskas skrivas över är applikationens nya assemblies, dessa ska ju inte ersättas med de gamla igen)

XSD / XML / CLASS
    Utility to generate schema or class files from given source.

xsd.exe <schema>.xsd /classes|dataset [/e:] [/l:] [/n:] [/o:] [/s] [/uri:]
xsd.exe <assembly>.dll|.exe [/outputdir:] [/type: [...]]
xsd.exe <instance>.xml [/outputdir:]
xsd.exe <schema>.xdr [/outputdir:]
Kör VS Command prompt
För hjälp: xsd /?

Filnamn på cs-fil blir samma som xsd-filnamnet
xsd [sökväg och xsd-schema] /classes /language:cs /namespace:[önskad språk output ska vara i] /out:[sökväg till katalog filen]

ex.
xsd "C:\Program Files\OSGeo\MapGuide\Server\Schema\LayerDefinition-1.0.0.xsd" /classes /language:cs /namespace:Plan.Plandokument /out:"\\landskrona.se\dfs\users\chran7\Documents\Visual Studio 2012\Projects\plan\class" /nologo

LÖSNING:
Koruppt installation av asp.net eller version av .net framework.
Vid t.ex. 32-bitars maskin och version 4 eller 4.5 av .net framework

1) Öppna kommandotolken (cmd) som administratör
2) Reperation av .net framework
    a) från katalog
       cd\
       cd "Windows\Microsoft.NET\Framework\v4.0.30319\SetupCache"
    b) kör
       setup.exe /repair /x86 /x64 /ia64 /norestart

    Om datorn kör en klientprofil av .net framework gäller följande istället
    a) från katalog
       cd\
       cd "Windows\Microsoft.NET\Framework\v4.0.30319\SetupCache\Client"
    b) kör
       setup.exe /repair /x86 /x64 /ia64 /parameterfolder Client /norestart
3) från katalog installera asp.net version som assossieras med .net versionen
   cd\
   cd "Windows\Microsoft.NET\Framework\v4.0.30319\SetupCache"
   aspnet_regiis.exe /i
4) starta och stoppa hela webbservern och tömmer alla applikationspooler
   iisreset
